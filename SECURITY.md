# SECURITY.md — Credentials and Secrets

## RULES (LOCKED)

1. **NEVER put secrets in this repo.** Not in code, not in docs, not in prompts committed as files, not in markdown examples. Not even in private commits — git history is permanent.
2. **NEVER include a PAT, API key, password, or any credential in a chat prompt or agent spawn message.** Secrets in prompts leak into chat history, agent execution traces, and screen captures.
3. **PATs live in `~/.git-credentials` on each agent runtime**, set up once manually by Rain on each machine, never by an agent.
4. **PATs must be fine-grained, repo-scoped, and ≤7-day expiry.** Classic PATs are over-scoped; do not use them.
5. **If you (an agent) hit a git auth error, REPORT IT.** Do not ask for a PAT in chat. Rain re-runs the one-time setup on the runtime.

## One-time setup (run manually on each agent runtime, never through Claude)

```bash
git config --global credential.helper store
echo "https://dvaneetv:YOUR_NEW_FINE_GRAINED_PAT@github.com" > ~/.git-credentials
chmod 600 ~/.git-credentials
git config --global user.email "dvaneetv@ucsd.edu"
git config --global user.name "claude_agent"
```

## Spawn prompt SETUP block (the only permissible form)
SETUP:
git clone https://github.com/beepbeeepimajeep/151B_SP26_Competition.git /home/claude/repo
cd /home/claude/repo
Credentials are pre-configured in ~/.git-credentials by Rain.
If git push fails with auth error, REPORT to Rain — do NOT request a PAT in chat.

## When something leaks

1. Immediately revoke at GitHub → Settings → Developer settings → Personal access tokens → Revoke
2. Generate a replacement (fine-grained, 7-day, repo-scoped)
3. Re-run the one-time setup on each agent runtime
4. Log the incident in the "Past incidents" section below
